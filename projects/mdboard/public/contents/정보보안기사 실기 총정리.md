## 1. 시스템 보안

### 1-1. 운영체제 보안 (리눅스/유닉스)

#### 파일 권한
```
rwxrwxrwx = 777
-rw-r--r-- = 644 (일반 파일)
drwxr-xr-x = 755 (디렉터리)
```
- **SetUID**: 실행 시 소유자 권한으로 실행 (`chmod 4755`)
- **SetGID**: 실행 시 그룹 권한으로 실행 (`chmod 2755`)
- **Sticky Bit**: 소유자만 삭제 가능 (`chmod 1777`, `/tmp`)

#### 주요 파일
| 파일 | 내용 |
|------|------|
| `/etc/passwd` | 사용자 계정 정보 (패스워드 x 표시) |
| `/etc/shadow` | 암호화된 패스워드 저장 |
| `/etc/group` | 그룹 정보 |
| `/etc/sudoers` | sudo 권한 설정 |
| `/var/log/auth.log` | 인증 로그 |
| `/var/log/syslog` | 시스템 로그 |
| `/var/log/secure` | 보안 관련 로그 (RHEL계열) |

#### 주요 명령어
```bash
# 계정/권한
useradd, usermod, userdel
chmod, chown, chgrp
passwd -l [user]    # 계정 잠금
passwd -u [user]    # 계정 잠금 해제

# 프로세스
ps aux             # 전체 프로세스
top / htop         # 실시간 모니터링
kill -9 [PID]      # 강제 종료
lsof -i            # 열린 파일/네트워크

# 네트워크
netstat -anp       # 포트 및 연결 확인
ss -tulnp          # 소켓 상태
ifconfig / ip addr # 네트워크 인터페이스
iptables           # 방화벽 규칙

# 로그/감사
last               # 로그인 기록
lastb              # 실패한 로그인
who, w             # 현재 로그인 사용자
find / -perm -4000 # SetUID 파일 검색
```

#### PAM (Pluggable Authentication Module)
- 설정 파일: `/etc/pam.d/`
- 4가지 모듈 유형: `auth`, `account`, `session`, `password`
- 제어 플래그: `required`, `requisite`, `sufficient`, `optional`

#### 주요 보안 설정
```bash
# /etc/login.defs 주요 설정
PASS_MAX_DAYS  90      # 패스워드 최대 사용 기간
PASS_MIN_DAYS  1       # 패스워드 최소 사용 기간
PASS_MIN_LEN   8       # 패스워드 최소 길이
PASS_WARN_AGE  7       # 만료 경고일
```

### 1-2. 윈도우 보안

#### 주요 레지스트리
```
HKLM\SAM           - 사용자 계정 정보
HKLM\SECURITY      - 보안 정책
HKLM\SOFTWARE      - 소프트웨어 설정
HKLM\SYSTEM        - 시스템 설정
HKCU               - 현재 사용자 설정
```

#### 주요 이벤트 로그 ID
| ID | 내용 |
|----|------|
| 4624 | 로그온 성공 |
| 4625 | 로그온 실패 |
| 4634 | 로그오프 |
| 4648 | 명시적 자격증명으로 로그온 |
| 4720 | 사용자 계정 생성 |
| 4722 | 사용자 계정 활성화 |
| 4740 | 계정 잠금 |
| 4776 | 자격증명 유효성 검사 |

#### SAM (Security Account Manager)
- 사용자 패스워드 해시 저장
- **NTLM 해시**: `MD4(Unicode(password))`
- **LM 해시**: 취약 (DES 기반, 대소문자 구분 없음) → 현재 비활성화 권장

#### 보안 정책
- **계정 잠금 정책**: 임계값, 기간, 카운터 재설정 시간
- **감사 정책**: 로그온, 개체 액세스, 정책 변경, 권한 사용 등
- **사용자 권한 할당**: 네트워크에서 이 컴퓨터 액세스 등

## 2. 네트워크 보안

### 2-1. 네트워크 공격 유형

#### DoS / DDoS 공격
| 공격 | 설명 | 대응 |
|------|------|------|
| **SYN Flood** | TCP 3way handshake 미완료, 연결큐 고갈 | SYN Cookie, 연결 타임아웃 축소 |
| **UDP Flood** | 대량 UDP 패킷으로 대역폭 소진 | 방화벽 UDP 제한 |
| **ICMP Flood (Ping Flood)** | 대량 ICMP Echo 요청 | ICMP 제한 |
| **Smurf** | IP Broadcast + ICMP, 증폭 공격 | Directed Broadcast 차단 |
| **Teardrop** | 잘못된 IP Fragment Offset | 패킷 재조립 검증 |
| **Land** | 출발지=목적지 IP로 패킷 전송 | 방화벽 차단 |
| **Slow HTTP** | 느린 HTTP 요청으로 연결 점유 | 연결 타임아웃 설정 |
| **HTTP Flood** | 정상 HTTP 요청 대량 발생 | CAPTCHA, Rate Limiting |

#### 스캐닝 공격
| 스캔 | TCP 플래그 | 열린 포트 | 닫힌 포트 |
|------|-----------|---------|---------|
| **TCP Connect** | SYN → SYN/ACK | 연결 완성 | RST |
| **SYN (Half-Open)** | SYN → SYN/ACK | RST 전송 | RST |
| **FIN** | FIN | 응답 없음 | RST |
| **NULL** | 플래그 없음 | 응답 없음 | RST |
| **XMAS** | FIN+URG+PSH | 응답 없음 | RST |
| **ACK** | ACK | RST | RST (방화벽 여부 판단) |
| **UDP** | - | 응답 없음 | ICMP Unreachable |

#### ARP 관련 공격
- **ARP Spoofing**: 위조 ARP Reply로 MAC 테이블 오염 → MITM
- **ARP Flooding**: 대량 ARP 패킷으로 스위치 혼란 → 허브 모드 전환
- **대응**: ARP 테이블 정적 설정, Dynamic ARP Inspection (DAI)

#### IP 스푸핑
- 공격자가 출발지 IP 위조
- 일반적으로 TCP에서는 어려움 (3way handshake)
- **ingress/egress 필터링**으로 대응

#### 세션 하이재킹
- TCP 시퀀스 번호 예측하여 세션 탈취
- **대응**: 암호화(TLS), 세션 타임아웃, 랜덤 ISN

---

### 2-2. 스니핑 (Sniffing)

- 네트워크 패킷 도청
- **수동적 스니핑**: 허브 환경 (무차별 모드)
- **능동적 스니핑**: 스위치 환경 (ARP Spoofing, MAC Flooding)

**대응**: 스위치 사용, 암호화(TLS/SSL), VPN

---

### 2-3. 방화벽 (Firewall)

#### 유형
| 유형 | 특징 | 계층 |
|------|------|------|
| **패킷 필터링** | IP/Port 기반, 빠름, 상태 없음 | 3계층 |
| **상태 기반 (Stateful)** | 연결 상태 추적 | 3~4계층 |
| **애플리케이션 게이트웨이** | 프록시 방식, 내용 검사 | 7계층 |
| **서킷 게이트웨이** | 세션 레벨, SOCKS | 5계층 |
| **차세대 방화벽 (NGFW)** | DPI + IPS + App 인식 | 7계층 |

#### 구성 방식
- **스크리닝 라우터**: 라우터에서 패킷 필터링
- **배스천 호스트**: 강화된 단일 호스트
- **DMZ (비무장지대)**: 내부/외부 사이 완충 영역
- **Dual-Homed Gateway**: 라우팅 비활성화된 방화벽
- **Screened Subnet**: DMZ + 스크리닝 라우터 (가장 안전)

---

### 2-4. IDS / IPS

| 구분 | IDS | IPS |
|------|-----|-----|
| 배치 | 네트워크 외부/내부 모니터링 | 인라인 (트래픽 경로상) |
| 역할 | 탐지 및 알림 | 탐지 + 차단 |
| 오탐 영향 | 낮음 | 높음 (정상 차단 위험) |

#### 탐지 방식
- **시그니처 기반 (오용 탐지)**: 알려진 패턴 매칭 → 오탐 낮음, 미탐 높음
- **이상 탐지 (Anomaly)**: 정상 기준선 대비 이상 감지 → 오탐 높음, 미탐 낮음
- **하이브리드**: 두 방식 조합

#### 오류 유형
- **False Positive (FP)**: 정상을 공격으로 탐지 (오탐)
- **False Negative (FN)**: 공격을 정상으로 탐지 (미탐)

---

### 2-5. VPN

| 유형 | 계층 | 프로토콜 |
|------|------|---------|
| **IPSec VPN** | 3계층 | AH, ESP |
| **SSL VPN** | 7계층 | TLS |
| **L2TP** | 2계층 | L2TP + IPSec |
| **PPTP** | 2계층 | 취약 (비권장) |

#### IPSec 구성 요소
- **AH (Authentication Header)**: 무결성 + 인증 (암호화 없음)
- **ESP (Encapsulating Security Payload)**: 기밀성 + 무결성 + 인증
- **IKE**: 키 교환 및 SA 협상
- **SA (Security Association)**: 단방향 보안 세션

**동작 모드**:
- **전송 모드**: IP 헤더 보존, 페이로드만 보호 (종단간)
- **터널 모드**: 원본 패킷 전체 캡슐화 (게이트웨이간)

---

### 2-6. DNS 보안

- **DNS Spoofing**: 위조 DNS 응답으로 잘못된 IP 제공
- **DNS Hijacking**: DNS 서버 탈취
- **DNS Amplification**: 소형 쿼리로 대형 응답 유발 (DDoS)
- **DNSSEC**: DNS 응답에 전자서명으로 무결성 보장

| 프로토콜 | 암호화 | 취약점 |
|---------|------|------|
| **WEP** | RC4 | 매우 취약 (IV 재사용) |

## 3. 애플리케이션 보안

### 3-1. 웹 취약점 (OWASP Top 10)

#### SQL 인젝션
```sql
-- 공격 예시
' OR '1'='1
' OR 1=1--
'; DROP TABLE users;--
' UNION SELECT null, username, password FROM users--
```
**대응**: 준비된 구문(Prepared Statement), 입력값 검증, WAF

#### XSS (Cross-Site Scripting)
| 유형 | 설명 |
|------|------|
| **Reflected XSS** | URL 파라미터에 스크립트 삽입, 즉시 실행 |
| **Stored XSS** | DB에 저장 후 출력 시 실행 |
| **DOM-based XSS** | 클라이언트 DOM 조작 |

```html
<!-- 공격 예시 -->
<script>document.cookie</script>
<img src=x onerror=alert(1)>
```
**대응**: 출력 인코딩(HTML Entity), CSP, HttpOnly 쿠키

#### CSRF (Cross-Site Request Forgery)
- 인증된 사용자의 세션을 이용한 위조 요청
- **대응**: CSRF 토큰, SameSite 쿠키, Referer 검증

#### 파일 업로드 취약점
- 악성 파일(.php, .jsp 등) 업로드 후 실행
- **대응**: 확장자/MIME 타입 검증, 저장 경로 분리, 실행 권한 제거

#### 파일 다운로드(Path Traversal)
```
../../../etc/passwd
..%2F..%2F..%2Fetc%2Fpasswd
```
**대응**: 경로 정규화, 허용 경로 화이트리스트

#### 커맨드 인젝션
```
; cat /etc/passwd
| whoami
&& rm -rf /
```
**대응**: 입력값 검증, 쉘 사용 자제, 화이트리스트

#### 디렉토리 리스팅
- 웹서버 디렉토리 목록 노출
- **대응**: `Options -Indexes` (Apache)

#### 인증/인가 취약점
- **Broken Authentication**: 세션 ID 취약, 브루트포스 허용
- **Broken Access Control**: 수평적/수직적 권한 상승
- **IDOR**: 직접 객체 참조 (예: `?id=123` → `?id=124`)

---

### 3-3. 쿠키 보안 속성

```
Set-Cookie: session=abc; HttpOnly; Secure; SameSite=Strict; Path=/
```
- **HttpOnly**: JavaScript 접근 차단 (XSS 방어)
- **Secure**: HTTPS에서만 전송
- **SameSite**: CSRF 방어 (`Strict`, `Lax`, `None`)

---

### 3-4. HTTPS / TLS

#### TLS Handshake (1.2)
```
Client Hello → Server Hello → Certificate → Server Hello Done
Client Key Exchange → Change Cipher Spec → Finished
Change Cipher Spec → Finished
```

#### TLS 취약점
| 취약점 | 설명 |
|--------|------|
| **POODLE** | SSL 3.0 취약점 |
| **BEAST** | TLS 1.0 CBC 취약점 |
| **CRIME/BREACH** | 압축 + 암호화 사이드채널 |
| **Heartbleed** | OpenSSL 버퍼 오버리드 |
| **DROWN** | SSLv2 허용 시 TLS 복호화 |

**대응**: TLS 1.2 이상 사용, 취약 암호 스위트 제거

---

### 3-5. 소프트웨어 개발 보안 (SDLC)

#### 보안 개발 방법론
- **MS SDL (Security Development Lifecycle)**: 요구사항~운영 전 단계 보안
- **CLASP**: 활동 기반 보안 프로세스
- **Seven Touchpoints**: 소프트웨어 보안 우수 사례
- **OWASP SAMM**: 성숙도 모델

#### 시큐어 코딩 원칙
- 입력값 검증 (화이트리스트 우선)
- 최소 권한 원칙
- 기본값 안전 (Fail-Secure)
- 오류 처리 (상세 오류 메시지 미노출)
- 암호화 통신

## 4. 데이터베이스 보안

### 4-1. 데이터베이스 보안 위협

| 위협 | 내용 |
|------|------|
| **SQL 인젝션** | 악의적 쿼리 삽입 |
| **권한 남용** | 과도한 권한 부여 |
| **미패치** | DB 취약점 방치 |
| **약한 인증** | 기본 계정/패스워드 |
| **감사 부재** | 접근 로그 없음 |
| **데이터 유출** | 덤프, 과도한 조회 |

### 4-2. 데이터베이스 보안 대책

- **접근 제어**: 최소 권한 원칙, 역할(Role) 기반
- **암호화**: 컬럼 암호화, TDE(Transparent Data Encryption)
- **감사 로깅**: DML/DDL/접속 이력 기록
- **DAM (DB Activity Monitoring)**: 실시간 DB 접근 모니터링
- **DB 방화벽**: SQL 트래픽 필터링

### 4-3. 개인정보 보호

- **가명처리**: 식별 정보를 다른 값으로 대체 (재식별 가능)
- **익명처리**: 재식별 불가 처리
- **데이터 마스킹**: 특정 필드 숨김/변환
  - 정적 마스킹: 복사본에 적용
  - 동적 마스킹: 조회 시 실시간 적용

## 5. 암호학

### 5-1. 암호화 방식 비교

| 구분 | 대칭키 | 비대칭키 |
|------|--------|---------|
| 키 수 | 1개 (동일 키) | 2개 (공개키/개인키) |
| 속도 | 빠름 | 느림 |
| 키 관리 | 어려움 (n(n-1)/2) | 용이 (2n) |
| 용도 | 데이터 암호화 | 키 교환, 전자서명 |
| 알고리즘 | AES, DES, 3DES, SEED, ARIA | RSA, ECC, ElGamal, DSA |

### 5-2. 주요 대칭키 알고리즘

| 알고리즘 | 블록 크기 | 키 길이 | 특징 |
|---------|---------|--------|------|
| **DES** | 64bit | 56bit | 취약 (사용 금지) |
| **3DES** | 64bit | 112/168bit | DES 3회 적용 |
| **AES** | 128bit | 128/192/256bit | 현재 표준 |
| **SEED** | 128bit | 128bit | 국내 표준 |
| **ARIA** | 128bit | 128/192/256bit | 국내 표준 (NIST 공모) |
| **Blowfish** | 64bit | 32~448bit | 가변 키 |

### 5-3. 블록 암호 운영 모드

| 모드 | 이름 | 특징 |
|------|------|------|
| **ECB** | 전자 코드북 | 같은 평문 → 같은 암호문 (취약) |
| **CBC** | 암호 블록 연결 | XOR 체이닝, IV 필요 |
| **CTR** | 카운터 | 스트림처럼 사용, 병렬 가능 |
| **CFB** | 암호 피드백 | 스트림 암호처럼 동작 |
| **OFB** | 출력 피드백 | 키스트림 사전 생성 가능 |
| **GCM** | 갈루아/카운터 | 인증 포함, AEAD |

### 5-4. 해시 함수

| 알고리즘 | 출력 길이 | 상태 |
|---------|---------|------|
| **MD5** | 128bit | 충돌 취약 (사용 금지) |
| **SHA-1** | 160bit | 취약 (단계적 폐지) |
| **SHA-256** | 256bit | 현재 권장 |
| **SHA-384** | 384bit | 강화 |
| **SHA-512** | 512bit | 강화 |
| **SHA-3** | 가변 | 새로운 표준 |

**해시 특성**: 일방향성, 충돌 저항성, 눈사태 효과, 역상 저항성

### 5-5. 비대칭키 알고리즘

| 알고리즘 | 기반 수학 | 용도 |
|---------|---------|------|
| **RSA** | 소인수분해 어려움 | 암호화, 서명 |
| **ECC** | 타원곡선 이산로그 | 짧은 키로 동등 보안 |
| **DSA** | 이산로그 | 전자서명 전용 |
| **ElGamal** | 이산로그 | 암호화, 서명 |
| **DH** | 이산로그 | 키 교환 |

### 5-6. 전자서명

```
[서명 생성]  해시(메시지) → 개인키로 암호화 → 서명값
[서명 검증]  서명값을 공개키로 복호화 → 해시 비교
```

**제공 서비스**: 무결성, 인증, 부인방지

### 5-7. PKI (공개키 기반구조)

구성 요소:
- **CA (인증기관)**: 인증서 발급/폐기
- **RA (등록기관)**: 신원 확인 후 CA에 요청
- **VA (검증기관)**: 인증서 유효성 검증
- **저장소**: CRL, OCSP 등 배포

**X.509 인증서** 포함 내용: 버전, 일련번호, 서명 알고리즘, 발급자, 유효기간, 주체, 공개키, CA 서명

**인증서 폐지 확인**:
- **CRL (Certificate Revocation List)**: 폐지 목록 주기적 배포
- **OCSP (Online Certificate Status Protocol)**: 실시간 온라인 확인

## 6. 접근통제

### 6-1. 접근통제 모델

| 모델 | 특징 | 적용 |
|------|------|------|
| **DAC** (임의적) | 소유자가 권한 결정, 유연 | 일반 파일 시스템 |
| **MAC** (강제적) | 보안 레이블 기반, 중앙 통제 | 군사, 정부 |
| **RBAC** (역할기반) | 역할에 권한 부여 | 기업 환경 |
| **ABAC** (속성기반) | 속성 조합으로 동적 제어 | 클라우드, 현대 시스템 |

### 6-2. MAC 보안 모델

| 모델 | 규칙 | 목적 |
|------|------|------|
| **Bell-LaPadula** | No Read Up, No Write Down | 기밀성 |
| **Biba** | No Read Down, No Write Up | 무결성 |
| **Clark-Wilson** | 트랜잭션 무결성 | 무결성 |
| **Chinese Wall** | 이해충돌 방지 | 기밀성 + 무결성 |

### 6-3. 인증 방식

**3가지 인증 요소**:
1. **지식 기반** (What you know): 패스워드, PIN
2. **소유 기반** (What you have): OTP, 스마트카드
3. **존재 기반** (What you are): 지문, 홍채, 얼굴

**MFA (다요소 인증)**: 2가지 이상 조합

#### 생체인증 오류율
- **FAR (False Acceptance Rate)**: 타인 수락률 (낮을수록 보안 강함)
- **FRR (False Rejection Rate)**: 본인 거부율 (낮을수록 편의성 좋음)
- **EER (Equal Error Rate)**: FAR = FRR 지점 → 낮을수록 좋음

### 6-4. OTP (One-Time Password)

| 방식 | 기반 | 예시 |
|------|------|------|
| **HOTP** | 이벤트(카운터) | 버튼 누를 때마다 |
| **TOTP** | 시간 (30초) | Google Authenticator |
| **S/KEY** | 해시 체인 | |

#### 주요 기법
```bash
# 디스크 이미징
dd if=/dev/sda of=disk.img bs=512

# 해시 검증
md5sum disk.img
sha256sum disk.img

# 휘발성 데이터 수집 순서 (Order of Volatility)
1. CPU 레지스터, 캐시
2. 메모리 (RAM)
3. 네트워크 상태
4. 실행 중 프로세스
5. 디스크
6. 로그
7. 아카이브
```

- **브라우저 히스토리**: 각 브라우저 경로
- **썸네일 캐시**: `thumbs.db`, `thumbcache_*.db`
- **Shellbags**: 폴더 접근 기록
- **LNK 파일**: 최근 접근 파일 링크

### 8-2. 위험 관리

#### 위험 분석 방법
| 방법 | 특징 |
|------|------|
| **정량적** | 금액으로 표현, ALE = SLE × ARO |
| **정성적** | 등급/점수로 표현, 주관적 |
| **기준선 접근법** | 표준 통제 적용, 빠름 |
| **복합 접근법** | 정량 + 정성 혼합 |

**주요 용어**:
- **자산(Asset)**: 보호 대상
- **위협(Threat)**: 자산에 손해를 줄 수 있는 행위
- **취약점(Vulnerability)**: 위협이 이용하는 약점
- **위험(Risk)**: 위협 × 취약점 × 자산가치
- **위험 수용(Risk Acceptance)**: 위험 감수
- **위험 회피(Risk Avoidance)**: 위험 요소 제거
- **위험 전가(Risk Transfer)**: 보험, 외주
- **위험 완화(Risk Mitigation)**: 통제 적용

**정량적 분석 공식**:
```
SLE (단일 손실 예상액) = 자산가치 × 노출계수(EF)
ALE (연간 손실 예상액) = SLE × ARO (연간 발생률)
```

### 8-5. BCP / DRP

**BCP (업무 연속성 계획)**: 재해 상황에서도 업무 지속  
**DRP (재해 복구 계획)**: IT 시스템 복구에 초점

**주요 지표**:
- **RTO (복구 목표 시간)**: 서비스 복구까지 최대 허용 시간
- **RPO (복구 목표 시점)**: 허용 가능한 최대 데이터 손실 시점
- **MTBF (평균 장애 간격)**: 장애와 장애 사이 평균 시간
- **MTTR (평균 복구 시간)**: 장애 발생 후 복구까지 평균 시간

**백업 복구 센터 유형**:
| 유형 | 특징 | RTO |
|------|------|-----|
| **핫 사이트** | 즉시 전환 가능, 비용 高 | 수 시간 이내 |
| **웜 사이트** | 부분 구성, 중간 비용 | 수 일 |
| **콜드 사이트** | 공간만 확보, 비용 低 | 수 주 |
