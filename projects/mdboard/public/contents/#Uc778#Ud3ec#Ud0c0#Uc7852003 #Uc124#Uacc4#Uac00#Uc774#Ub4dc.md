# IT2003 (근무조변경) 하위유형 설계 가이드

## 유연근무(시차출퇴근) / 단축근무 구현



> 작성일 : 2026-04-15  

> 대상 시스템 : SAP HCM (PA / PT 모듈)



---



## 1. 구현 전략 개요



### 기본 설계 원칙



근무일정(Work Schedule Rule)을 케이스별로 대량 생성하면  

**`공휴일달력 × 유연근무 케이스 × 기간`** 조합으로 관리 부담이 폭증한다.



이를 피하기 위해 아래 구조를 채택한다.



```

IT0007 (기본 근무일정)

&#x20; └─ 표준 WSR 1개 유지 (09:00~18:00 기준)



IT2003 (근무조변경)

&#x20; └─ 개인별 시차 / 단축 케이스를 날짜 범위로 Override

&#x20;      → WSR 추가 없이 DWS 교체만으로 처리

```



### 핵심 장점



- WSR은 1개 유지 → 공휴일 달력 연결 중복 없음

- DWS만 케이스별 5~10개 생성으로 전체 커버

- IT2003 기간 설정으로 개인별 유연한 적용 가능

```
&#x20;    └─ S3 : 단축 4H 오후 (13:00~17:00)

```

## 3. 일일 WS 클래스 (Daily Work Schedule Class)



### 개념



DWS에 설정하는 **분류 코드**로, Time Evaluation 스키마가  

해당 날을 어떤 성격의 근무일로 처리할지 판단하는 기준값.



> "오늘이 어떤 날인지 Time Evaluation에게 알려주는 태그"



### 표준 클래스 값



| Class | 의미 | 비고 |

|---|---|---|

| **0** | 비근무일 (Off day) | 휴일, 무급휴무일 |

| **1** | 정상 근무일 | 일반 평일 |

| **2** | 근무 있는 특수일 | 단축근무, 교대 특수일 |

| **3** | 출퇴근 기록 없는 근무일 | 재량근무, 간주근로 |

| **9** | 공휴일 대체 처리 | Holiday Calendar 연동 시 자동 부여 |



### 유연근무 / 단축근무 권장 Class 값



| 구분 | 권장 Class | 이유 |

|---|---|---|

| **유연근무 (시차출퇴근)** | **1** | 총 근무시간 동일(8H), 정상 근무일과 동일 처리 |

| **단축근무** | **2** | 소정시간 감소, 특수 근무일임을 스키마에 알려야 함 |

| 재량근무 / 간주근로 | 3 | 출퇴근 기록 없이 소정시간 자동 인정 |

| 비근무일 | 0 | 근무 계획 자체 없음 |



### DWS Class 확인 방법



**SE16N → T550A 조회**



| 필드명 | 설명 |

|---|---|

| MOREI | DWS 코드 |

| **TKLAS** | **DWS Class ← 핵심** |

| BEGUZ | 근무 시작시간 |

| ENDUZ | 근무 종료시간 |

| ARBST | 소정근무시간 |



```

커스텀 DWS만 필터 : MOREI 필드에 Z* 입력

```



또는 `SPRO > Time Management > Work Schedules > Daily Work Schedules > Define Daily Work Schedules` 에서 DWS 선택 후 상세 화면의 "DWS Class" 필드 확인.

## 4. Substitution Type 세팅 (T556A)



### 설정 경로



```

SPRO

└─ Time Management

&#x20;  └─ Work Schedules

&#x20;     └─ Substitutions

&#x20;        └─ Define Substitution Types

```



### 유연근무 (Z1~Z4) T556A 설정



| 항목 | 설정값 | 이유 |

|---|---|---|

| Substitution Type | Z1 ~ Z4 | 시차 케이스별 구분 |

| Short text | 시차07, 시차08 등 | 화면 표시명 |

| **Time Substitution** | **✔ 체크** | 시간대 변경이므로 필수 |

| Day Substitution | 공백 | 날짜 자체는 유지 |

| **Planned hours change** | **공백** | 총 근무시간(8H) 동일 유지 |



### 단축근무 (S1~S3) T556A 설정



| 항목 | 설정값 | 이유 |

|---|---|---|

| Substitution Type | S1 ~ S3 | 단축 케이스별 구분 |

| Short text | 단축6H, 단축4H오전 등 | 화면 표시명 |

| **Time Substitution** | **✔ 체크** | 시간대 변경 |

| Day Substitution | 공백 | |

| **Planned hours change** | **✔ 체크** | **소정시간 자체가 줄어듦 → 필수** |



### 유연 vs 단축 핵심 차이



```

유연근무  →  총 8H 동일, 시작/종료 시간만 이동

&#x20;            ∴ Planned hours change = 공백



단축근무  →  총 근무시간 자체 감소 (4H, 6H)

&#x20;            ∴ Planned hours change = ✔ 체크 (필수)

```

## 5. 개별 근무 시간 활용 방식



IT2003 레코드 입력 시 두 가지 방법 중 선택:



### 방법 A : DWS 직접 지정 (권장)



- IT2003에서 해당 시차/단축 DWS 코드를 선택

- DWS에 시작/종료/코어타임이 미리 정의되어 있어 일관성 유지

- 오입력 리스크 최소화



### 방법 B : 시간 직접 입력 (개별 시간 활용)



- IT2003 레코드에 시작시간 / 종료시간 수기 입력

- DWS를 만들지 않고 개인별로 다른 시간 적용 가능

- 유연하나 **오입력 위험 존재**, 보고서 집계 불편



> **권장:** 표준 케이스는 DWS 지정, 예외적 케이스만 시간 직접 입력

## 7. Time Evaluation 처리



### 유연근무



```

IT2003 존재하는 날

&#x20; → DWS Override → 해당 시차 계획시간으로 재계산

&#x20; → 실제 출퇴근(IT2002)과 비교

&#x20; → 총 8H 충족 시 초과근무 미발생으로 PCR 설계 권장

&#x20; → 코어타임 미충족 시 경고 또는 결근 처리 PCR 별도 추가

```



### 단축근무



```

IT2003 존재하는 날

&#x20; → Planned hours change로 소정시간 감소 반영

&#x20; → 감소된 소정시간 기준으로 출퇴근 비교

&#x20; → 급여 감액 Wage Type 자동 생성 여부 → Payroll 스키마 연계

```

## 8. ⚠️ 주의사항



### 유연근무 관련



**① 코어타임 강제 기능 없음**

IT2003 자체는 코어타임을 강제하지 않는다.  

Time Evaluation PCR에서 별도 체크 로직을 반드시 구현해야 한다.



**② 공휴일 중복 시 처리**

IT2003 기간 중 공휴일이 포함된 날은 WSR의 Holiday Calendar가 우선 적용된다.  

IT2003 레코드가 살아있어도 공휴일로 정상 처리되나, 보고서에서 혼선이 생길 수 있으니 주의.



**③ 급여 정산 기간과 불일치**

월 중간에 시차 변경 시 급여 계산 기간과 어긋날 수 있다.  

→ 운영 원칙 : **월 초 기준으로 IT2003 적용** 권장



### 단축근무 관련



**④ Planned Hours 미체크 시 초과근무 오발생 (가장 치명적)**

`Planned hours change`를 체크하지 않으면 소정시간은 8H로 유지된 채  

4~6H만 근무하게 되어 **매일 결근/부족시간이 발생**한다.



**⑤ 급여 처리 기준 미사전 결정**

단축근무 시 급여 감액 여부를 IT2003 설정만으로 자동화할 수 없다.  

→ Payroll 스키마 + Wage Type 연계를 HR·급여팀이 **사전 협의** 필요



**⑥ 사회보험 기준 변동**

단축근무로 월 소정근로시간이 60H 미만이 되면 사회보험 가입 기준 변동 발생 가능.  

→ IT0021 및 사회보험 인포타입 연계 검토 필요



### 공통



**⑦ IT2003 vs IT2001 우선순위**

동일 날짜에 IT2003(근무조변경)과 IT2001(결근)이 동시 존재할 경우  

**IT2001(결근)이 우선** 적용된다. 팀 내 사전 공유 필요.



**⑧ 소급 변경 → Payroll Retro 트리거**

이미 급여가 확정된 기간에 IT2003을 소급 생성/변경하면 소급정산이 트리거된다.  

→ 운영 정책 : **전월 이전 소급 변경 금지** 원칙 수립 권장

## 10. 구현 체크리스트



### 유연근무 (Z*)



- [ ] T556A : Time Substitution ✔, Planned hours change 공백

- [ ] DWS 시차별 생성 (TKLAS = 1, 코어타임 포함)

- [ ] T591S / T591A : 하위유형 텍스트 등록

- [ ] Field Selection : 불필요 필드 Hidden 처리

- [ ] PCR : 코어타임 미충족 경고 로직 구현

- [ ] 권한 : ESS 직원 본인 신청 허용 여부 결정



### 단축근무 (S*)



- [ ] T556A : Time Substitution ✔, **Planned hours change ✔ (필수)**

- [ ] DWS 단축시간별 생성 (TKLAS = 2)

- [ ] T591S / T591A : 하위유형 텍스트 등록

- [ ] Field Selection : 단축사유 등 필드 구성

- [ ] 급여 감액 Wage Type 및 Payroll 스키마 연계 설계

- [ ] 사회보험 기준 검토

- [ ] 권한 : HR 담당자 전용



### 공통



- [ ] IT2001 중복 우선순위 운영 공지

- [ ] 소급 변경 금지 운영 정책 수립

- [ ] 월 초 기준 IT2003 적용 원칙 수립

- [ ] PT80으로 IT2003 Override 정상 동작 확인
