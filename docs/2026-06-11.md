## 5. GitHub 시크릿 스캔 차단 해결

### 문제
`projects/campchecklist/get-token.js`에 Google OAuth 키가 하드코딩된 채로 커밋(`42d1ba7`)에 포함됨.  
GitHub Push Protection이 푸시를 차단.

```
remote: - Push cannot contain secrets
remote:   —— Google OAuth Client ID ——  commit: 42d1ba7  path: get-token.js:21
remote:   —— Google OAuth Client Secret —— commit: 42d1ba7  path: get-token.js:22
```

### 조치
`GIT_SEQUENCE_EDITOR`를 이용한 자동화 interactive rebase로 해당 커밋만 `edit` 모드 지정 후 amend.

```bash
# 변경 사항 stash
git stash

# 42d1ba7 커밋만 edit 모드로 지정
GIT_SEQUENCE_EDITOR='sed -i "s/^pick 42d1ba7/edit 42d1ba7/"' git rebase -i HEAD~3

# get-token.js 시크릿 제거 후 amend
git add projects/campchecklist/get-token.js
git commit --amend --no-edit

git rebase --continue
git stash pop
git push origin main
```

**수정 내용** (`get-token.js` 21~22번째 줄):
```js
// 수정 전
const CLIENT_ID     = process.env.GDRIVE_CLIENT_ID     || '540709794004-...실제키...';
const CLIENT_SECRET = process.env.GDRIVE_CLIENT_SECRET || 'GOCSPX-...실제키...';

// 수정 후
const CLIENT_ID     = process.env.GDRIVE_CLIENT_ID     || '여기에 CLIENT_ID 입력';
const CLIENT_SECRET = process.env.GDRIVE_CLIENT_SECRET || '여기에 CLIENT_SECRET 입력';
```

> ⚠️ **재발 방지**: `get-token.js`는 토큰 발급용 1회성 스크립트. 실제 키는 환경변수로만 전달하고 코드에 하드코딩하지 않는다.

## 6. mdboard 백업 — GitHub → Google Drive 전환

### 배경
- 기존: `github.js` (GitHub Contents API)가 환경변수 미설정으로 동작 안 함
- 변경: CampChecklist와 동일한 OAuth2 인증을 공유하는 Google Drive 연동

### 신규 파일: `projects/mdboard/drive.js`

```
환경변수 (기존 CampChecklist 변수 공유):
  GDRIVE_CLIENT_ID
  GDRIVE_CLIENT_SECRET
  GDRIVE_REFRESH_TOKEN
  MDBOARD_FOLDER_ID    ← mdboard 전용 Drive 폴더 ID (신규 추가)
```

**기능:**

| 함수 | 동작 |
|---|---|
| `ensureInit()` | Lazy Init — 첫 파일 요청 시 Drive 연결 + 전체 복원 실행 |
| `pushFile(filename, content)` | 저장/수정 시 Drive에 즉시 upsert |
| `deleteFile(filename)` | 삭제 시 Drive에서도 제거 |
| `pullAll()` | Drive → 로컬 `.md` 파일 전체 복원 |

### Lazy Init 설계 (healthcheck 보호)

```
서버 기동
  └─ /health 요청 → 즉시 200 응답 ✅ (Drive 대기 없음)

첫 /mdboard/api/files 요청
  └─ ensureInit() 실행
       ├─ OAuth2 클라이언트 초기화
       ├─ Drive.files.list → .md 파일 목록
       ├─ 각 파일 다운로드 → public/contents/ 저장
       └─ 이후 요청은 즉시 통과
```

**미들웨어 배치 (index.js):**
```
router.get('/health')          ← Drive 대기 없음
router.get('/auth/check')      ← Drive 대기 없음
router.post('/auth')           ← Drive 대기 없음

router.use(drive.ensureInit)   ← 여기서부터 Drive 준비 대기

router.get('/files')
router.get('/file/:name')
router.post('/save')           → drive.pushFile()
router.delete('/file/:name')   → drive.deleteFile()
...
```

### Railway 환경변수 추가 필요
```
MDBOARD_FOLDER_ID = [Google Drive의 mdboard_contents 폴더 ID]
```
